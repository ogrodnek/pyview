How Streams Work:

  • ✅ Only send insert/delete operations, not the entire list
  • ✅ No server-side memory for maintaining the full collection
  • ✅ Client maintains the list and applies minimal diffs
  • ✅ Perfect for infinite scroll, live feeds, chat messages, etc.

Implementation Details

1. Initialize Stream in Context:

@dataclass
class StreamsContext:
    messages: Stream[Message]

messages_stream = Stream[Message](
    dom_id=lambda msg: f"msg-{msg.id}"
)

2. Mark Container in Template:

<div id="messages-stream"
     phx-update="stream"
     data-phx-stream="{{ context.messages.ref }}">
  {% for dom_id, msg in context.messages %}
    <div id="{{ dom_id }}">
      {{ msg.text }}
    </div>
  {% endfor %}
</div>

3. Perform Operations:

# Prepend
socket.context.messages.prepend(new_msg)

# Append
socket.context.messages.append(new_msg)

# Insert at position
socket.context.messages.insert(new_msg, at=5)

# Delete
socket.context.messages.remove(old_msg)
socket.context.messages.remove_by_id("msg-123")

# Reset
socket.context.messages.reset(new_list)
