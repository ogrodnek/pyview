How Streams Work

Streams allow efficient rendering of large lists. Instead of re-sending the entire list on each change, only the operations (insert, delete, reset) are sent over the websocket.

The "stream" key in the diff contains: [ref, inserts, deletes, reset?]

  • ref: Stream name (e.g., "messages")
  • inserts: Array of [dom_id, position, limit, update_only]
  • deletes: Array of DOM IDs to remove
  • reset: Boolean flag to clear all items first
